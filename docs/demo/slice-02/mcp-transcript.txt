=== initialize: instructions (Tier-1) ===
"Sidebar is a local-first markdown reading and co-authoring surface for a project's docs.\nIt hosts the project's workspace (markdown files matching the configured glob) and mediates\ncollaboration between the human (via a CodeMirror editor in the browser) and you (via this\nMCP server). Read tools surface the workspace; write tools, when present, are constrained.\n\nPermission model. You may read freely, but you may not edit prose unilaterally. Prose changes\nhappen in only two ways: (1) resolve_mention, when the human has placed a Mention marker\nauthorizing edits to a specific region; (2) add_annotation(type=suggestion), which proposes\nreplacement text that the human must accept. You can write note annotations anywhere for\ninformation; notes do not edit prose.\n\nbase_hash protocol. Mention lifecycle uses optimistic concurrency. When you read a mention\nyou receive its base_hash. resolve_mention must echo that hash. If the file changed since,\nyou get a conflict; refresh via get_mention and retry against the new content.\n\nis_draft signal. read_doc returns is_draft: bool and draft_age_seconds: int. When is_draft is\ntrue, the human has unsaved edits in the editor and the disk content you see is stale. Prefer\nto skip the file or create an agent-origin mention with verb clarify asking whether to proceed.\n\nFor full guidance (verb tables, mention round-trip examples, multi-agent etiquette, suggestion\nflow), run npx sidebar-md scaffold-skill in this project. That writes a skill file your agent can\nload alongside this server."
